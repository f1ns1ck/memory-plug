# Change Memory

[English](README.md) | **Русский**

> Change Memory даёт Claude Code компактную локальную память о том, что менялось в
> вашем проекте, между сессиями. Она хранит семантические сводки, изменённые
> файлы, нерешённые вопросы, риски и сжатые патчи — **не загружая полные диффы в
> контекст по умолчанию**.

**Категория:** development
**Ключевые слова:** claude-code, mcp, agent-memory, git-diff, context-management, token-optimization, coding-agent

---

## Что это делает

Долгие сессии кодинга жгут токены на перечитывание истории. Change Memory держит
крошечную «карту» изменений проекта на диске и отдаёт Claude **компактный снимок** в
начале сессии вместо полных диффов. Сами патчи хранятся локально, сжатые gzip, и
загружаются **только по явному запросу**.

Главный принцип:

> Агент получает короткую карту изменений, а не весь дифф. Полные файлы патчей
> загружаются только по явному запросу.

## Зачем это пользователям Claude Code

- **Непрерывность между сессиями** — продолжайте с того места, где остановились, без
  повторных объяснений.
- **Экономия токенов** — бутстрап-контекст ограничен бюджетом (~700 токенов по
  умолчанию).
- **Не повторять работу** — ищите прошлые фиксы, риски и нерешённые пункты.
- **Локально и приватно** — никакой сети, никакой телеметрии. Семантическую карту
  можно закоммитить и делиться с командой; сырые диффы (патчи) остаются на вашей
  машине.
- **Командно-дружелюбно** — история изменений путешествует вместе с репозиторием, с
  атрибуцией автора, так что следующий разработчик видит, кто, что и зачем менял.

## Как это работает

Плагин состоит из трёх частей:

1. **Agent Skill** (`skills/change-memory`) — учит Claude *когда* и *как*
   использовать память (прогрессивное раскрытие: сначала снимок, патчи — в последнюю
   очередь).
2. **MCP-сервер** (`mcp-server/`) — локальный Node.js-сервер, экспонирующий
   инструменты памяти.
3. **Slash-команды** (`commands/`) — быстрые точки входа.

Состояние лежит в вашем проекте под `.change-memory/`:

```
.change-memory/
  index.json       # метаданные проекта, активные файлы, нерешённые пункты, бюджеты  [shared]
  changes.jsonl    # по одной JSON-строке на захваченное изменение (вкл. автора)      [shared]
  summaries/       # архивная/уплотнённая история                                     [shared]
  session.md       # компактный снимок (зеркало get_session_context, пересоздаётся)   [local]
  patches/         # полные диффы под gzip (chg_*.patch.gz)                            [local]
  auto-capture.json# per-machine отпечаток + переключатель авто-захвата on/off         [local]
  .gitignore       # пишется init_memory: коммитит карту, игнорирует остальное
```

Файлы `[shared]` коммитятся, чтобы команда наследовала историю изменений; артефакты
`[local]` остаются на вашей машине. См. раздел **Командный рабочий процесс** ниже.

Смотрите [`examples/change-memory/`](examples/change-memory/) — статический пример
`session.md` и `index.json`, чтобы оценить формат без живой установки.

## Установка

1. Добавьте плагин из вашего маркетплейса Claude Code (или установите локально).
2. MCP-сервер запускается из скомпилированного вывода `mcp-server/dist/index.js`.
   - Если вы клонировали исходники, соберите один раз: `npm install` (скрипт
     `prepare` запускает `npm run build` автоматически). Требуется **Node.js ≥ 18**.
3. Перезапустите Claude Code, чтобы он подхватил встроенный MCP-сервер из `.mcp.json`.

### Включение MCP-сервера

`.mcp.json` регистрирует сервер для плагина:

```json
{
  "mcpServers": {
    "change-memory": {
      "command": "node",
      "args": ["${CLAUDE_PLUGIN_ROOT}/mcp-server/dist/index.js"]
    }
  }
}
```

`${CLAUDE_PLUGIN_ROOT}` предоставляется Claude Code и указывает на каталог
установленного плагина. Никакой конфигурации не требуется.

## Командный рабочий процесс

Change Memory создана, чтобы путешествовать вместе с репозиторием — чтобы **следующий
разработчик понимал, что и зачем менялось**, без ваших повторных объяснений.

- **Коммитьте карту.** `init_memory` пишет `.change-memory/.gitignore`, который
  коммитит `index.json`, `changes.jsonl` и `summaries/` (семантическую карту) и
  игнорирует `patches/`, `auto-capture.json` и `session.md` (машинно-локальное /
  тяжёлый бинарь). Просто коммитьте `.change-memory/` в рамках обычного git-потока —
  плагин сам никаких git-записей не делает.
- **Атрибуция.** Каждое изменение записывает `author` из `git config
  user.name/user.email`, так что `list_changes` и `show_change` показывают, *кто* его
  сделал.
- **Свежий клон.** Коллега, склонировавший репозиторий, запускает `/memory-session`
  (`get_session_context`) и сразу получает пересобранный снимок из закоммиченной
  карты — патчи не нужны. Он может смотреть метаданные любого изменения через
  `show_change`; сырой дифф (`includePatch: true`) доступен только для изменений,
  захваченных на его машине, так как патчи остаются локальными.
- **Делиться патчами тоже (opt-in).** Запустите `/memory share on` (или
  `set_share_patches({ enabled: true })`), чтобы коммитить и `patches/`, тогда команда
  сможет загружать дифф *любого* изменения. Флаг хранится как `share_patches` в
  закоммиченном `index.json`, и управляемый `.gitignore` регенерируется так, чтобы
  трекать `patches/`; `/memory share off` возвращает в режим «только локально».
  Включить можно и на старте через `/memory-init share`. Учтите: это коммитит сжатые
  патч-блобы, утяжеляя репозиторий — оставляйте выключенным, если команде не нужны
  полные диффы.

> **Не хотите ничем делиться?** Удалите сгенерированный `.change-memory/.gitignore` и
> добавьте `.change-memory/` в корневой `.gitignore` проекта, чтобы держать всё
> машинно-локально.

## Инструменты

| Инструмент | Назначение |
| --- | --- |
| `init_memory` | Создать `.change-memory/` для проекта. |
| `capture_change` | Снимок текущего `git diff` (вкл. untracked-файлы) → сжатый патч + семантическая сводка. |
| `auto_capture_change` | Как `capture_change`, но с дебаунсом и дедупликацией для автоматического (hook) использования. |
| `set_auto_capture` | Включить/выключить авто-захват для этой машины (per-developer; без `enabled` — запрос состояния). |
| `set_share_patches` | Включить/выключить шаринг патчей для проекта (командно через `index.json`; без `enabled` — запрос состояния). |
| `get_session_context` | Вернуть компактный markdown-снимок. **Никогда не включает полные диффы.** |
| `show_change` | Показать метаданные одного изменения; патч только при `includePatch: true`. |
| `list_changes` | Компактная таблица: `id \| type \| file \| summary`. |
| `search_changes` | Поиск по id, summary, files, reason, risk, tests. |
| `compact_memory` | Архивировать старые изменения в сводку; оставить недавние; сохранить патчи. |

## Slash-команды

Три основные команды — top-level; остальные живут под единым диспетчером `/memory`,
чтобы поверхность команд оставалась небольшой.

| Команда | Действие |
| --- | --- |
| `/memory-init [share-patches]` | Инициализировать память; добавьте `share`, чтобы коммитить и патчи. |
| `/memory-capture [reason]` | Захватить текущие изменения. |
| `/memory-session` | Загрузить компактный контекст сессии. |
| `/memory show <changeId> [patch]` | Показать изменение (добавьте `patch` для диффа). |
| `/memory search <query> [limit]` | Поиск по истории изменений по ключевому слову. |
| `/memory compact [olderThanDays] [keepRecent]` | Архивировать старые изменения; оставить недавние. |
| `/memory auto <on\|off\|status>` | Включить/выключить автоматический захват (per-machine). |
| `/memory share <on\|off\|status>` | Включить/выключить шаринг патчей (командно). |

## Автоматический захват

После установки плагина Change Memory записывает изменения **автоматически** — не
нужно запускать `/memory-capture` руками.

Встроенный хук Claude Code (`hooks/hooks.json`) срабатывает на `PostToolUse` для
инструментов `Write`, `Edit` и `MultiEdit` и вызывает MCP-инструмент
`auto_capture_change`.

- **Ручной захват по-прежнему работает.** `/memory-capture` (и инструмент
  `capture_change`) остаются доступны — используйте их для осознанного именованного
  снимка со своим reason.
- **С дебаунсом.** Не более одного захвата за окно `debounceMs` (по умолчанию
  **30с**), так что серия правок даёт один снимок, а не десятки.
- **С дедупликацией.** Авто-захват берёт отпечаток составленного диффа рабочего дерева
  (tracked-дифф + содержимое untracked-файлов). Если дифф не изменился с прошлого
  захвата — ничего не делает.
- **Никогда не трогает git-состояние.** Только читает `git diff` / `git status` и
  пишет в `.change-memory/`. Не делает `git add`, `commit`, `checkout` и никаких
  деструктивных git-команд.
- **Не блокирует.** Если память не инициализирована, путь не git-репозиторий или
  дерево чистое — хук тихо пропускает и не прерывает работу.

Авто-захват хранит свою бухгалтерию в `.change-memory/auto-capture.json`
(`last_fingerprint`, `last_capture_at`, `last_change_id`).

> **Про корень проекта:** хук передаёт `${CLAUDE_PROJECT_DIR}` (каталог, в котором был
> запущен Claude Code) как `projectPath`. Открывайте Claude Code **в корне git-репо**,
> а не в родительской папке — если репозиторий вложен ниже корня проекта, авто-захват
> нацелится на неверный каталог и тихо пропустит.

### Проверить, что авто-захват работает

1. Запустите `/memory-init` один раз в git-репо (до инициализации авто-захват — no-op).
2. Попросите Claude отредактировать файл. После правки проверьте последнюю запись:
   - `list_changes` (или `/memory show`) — новый `chg_...` с reason `auto: ...`.
   - `.change-memory/auto-capture.json` — обновился `last_change_id`.
3. Попросите вторую правку в пределах 30с → новой записи нет (дебаунс/дедуп). Подождите
   >30с и отредактируйте снова → появляется новая запись.

### Отключить авто-захват (режим только-вручную)

Самый простой способ — per-machine переключатель:

- Запустите **`/memory auto off`** (вызывает `set_auto_capture`). Флаг хранится в
  локальном, gitignored `auto-capture.json`, так что влияет только на вашу машину, не
  на коллег. `/memory auto on` включает обратно; `/memory auto status` сообщает текущее
  состояние.

Чтобы жёстко отключить хук для всех (или как запасной вариант):

- Удалите или переименуйте `hooks/hooks.json` в установленном плагине, **или**
- Отключите хуки плагина из `/hooks` / настроек хуков Claude Code, **или**
- Поднимите `debounceMs` в `hooks/hooks.json` до большого значения, чтобы задушить его.

MCP-инструменты (включая ручной `capture_change`) это не затрагивает.

## Пример рабочего процесса

```text
# Новая сессия
/memory-session          → Claude загружает компактный снимок

# ... вы и Claude вносите изменения ...

/memory-capture исправил обновление токена при истечении
                            → chg_20260618_210712_... | fix | Ada <ada@example.com> | src/auth.ts

# Позже вспоминаем детали
search_changes("token")     → находит изменение
show_change(chg_..., includePatch:false)  → только метаданные
show_change(chg_..., includePatch:true)   → полный дифф, по требованию
```

## Пример `.change-memory/session.md`

```md
# Session Context

Project: my-app

This is a compact memory snapshot for Claude Code.
It intentionally excludes full diffs to reduce token usage.

## Recent Changes

- chg_20260618_210712_de5c94c2 (Ada <ada@example.com>): Fix change: added 1,
  modified 1 file(s) in src (`src/auth.ts`). Reason: fix token refresh. Full
  patch stored locally.

## Active Files

- src/auth.ts

## Open Issues

- add tests for token expiry

## Constraints

- Keep context compact.
- Do not include full diffs by default.
- Load patch details only when needed.

## Available Memory Tools

- show_change(changeId)
- list_changes()
- search_changes(query)
- capture_change()
```

## Модель безопасности

Плагин рассчитан на безопасное распространение через маркетплейс:

1. **Только локально.** Всё состояние лежит в `.change-memory/` внутри вашего проекта.
2. **Без телеметрии**, без аналитики, без phone-home.
3. **Без внешних сетевых вызовов.** У сервера нет HTTP-клиента.
4. **Без `eval`** и без динамического исполнения кода.
5. **Без произвольного shell.** Единственный внешний процесс — `git`, ограниченный
   **allow-list-ом read-only векторов аргументов**:
   - `git diff`
   - `git diff --name-only`
   - `git diff --name-status`
   - `git status --porcelain [--untracked-files=all]`
   - `git rev-parse --is-inside-work-tree`
   - `git config user.name` / `git config user.email` (read-only, для атрибуции автора)

   Команды выполняются через `execFile` (без shell), и пользовательский ввод никогда не
   интерполируется в команду. Операции записи (commit/add/checkout) невозможны по
   конструкции.
6. **Никакой пользовательский код не меняется и не коммитится.** Захват — read-only.
7. **Защита от path traversal.** Каждый доступ к файлу нормализуется и проверяется на
   нахождение внутри корня проекта (`ensureInsideRoot`). Патчи живут только под
   `.change-memory/patches/`.
8. **Ограниченное чтение.** Содержимое untracked-файлов, встраиваемое в патчи,
   ограничено (256 КБ на файл), бинарные файлы пропускаются.

## Модель приватности

- **Сырые диффы никогда не покидают вашу машину.** Патчи хранятся сжатыми локально и
  по умолчанию gitignored; вы ими управляете.
- **Карта**, которую можно закоммитить (сводки, списки файлов, причины, авторы,
  нерешённые пункты), намеренно без диффов — только то, что нужно коллеге для
  понимания истории, а не сам исходник.
- Снимок сессии намеренно исключает диффы, чтобы минимизировать то, что попадает в
  контекст модели.
- Не хотите ничем делиться? См. **Командный рабочий процесс** — как держать весь
  каталог `.change-memory/` машинно-локальным.

## Ограничения MVP

Не входит в эту первую версию:

- Облачная синхронизация / удалённое хранилище
- Внешняя LLM-суммаризация (суммаризатор эвристический и офлайновый)
- Расширение VSCode или веб-дашборд
- Аутентификация
- Телеметрия
- Исполнение произвольных shell-команд
- Разделение staged-vs-unstaged (захватывается дифф рабочего дерева + untracked-файлы)

## Решение проблем

- **«No .change-memory found»** — запустите `/memory-init` (или вызовите
  `init_memory`).
- **«No changes to capture»** — в рабочем дереве нет tracked-изменений и нет
  untracked-файлов. Сначала внесите правку.
- **«Not a git repository»** — `capture_change` читает `git diff`; запускайте внутри
  git-репо (`git init`).
- **Сервер не загружается** — убедитесь, что `mcp-server/dist/index.js` существует;
  запустите `npm install` (он собирает) и перезапустите Claude Code.
- **Память захватила саму себя** — `.change-memory/` всегда исключается из захвата
  (см. `getUntrackedFiles`). Сгенерированный `.change-memory/.gitignore` уже держит
  патчи и машинно-локальное состояние вне коммитов.
- **Автор показывается как `(unknown)`** — задайте git-идентичность (`git config
  user.name` / `git config user.email`); у изменений, захваченных до этой версии,
  записанного автора нет.

## Дорожная карта (0.2.0+)

- Опциональный, opt-in LLM-суммаризатор (подключаемый интерфейс `Summarizer` уже на
  месте) для более богатых сводок — по-прежнему local-first.
- Получение патча по файлу в `show_change` (запрос ханка одного файла).
- Режимы захвата staged vs. unstaged и осведомлённость о ветке/коммите.
- Автоматический `compact_memory` по порогу размера/возраста.
- Настраиваемые бюджеты токенов и ограничения через хелперы редактирования
  `index.json`.
- Теги/метки и более богатое ранжирование поиска.
